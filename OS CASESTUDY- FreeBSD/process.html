Process Synchronization

Process Synchronization is the coordination of execution of multiple processes in a multi-process system to ensure that they access shared resources in a controlled and predictable manner. It aims to resolve the problem of race conditions and other synchronization issues in a concurrent system.

Irrigation System in Agriculture

Process Synchronization in Agricultural Irrigation System

One scenario in agriculture where process synchronization is crucial is irrigation management in a large farm with multiple fields. Let's consider a farm where there are multiple irrigation pumps (writers) and several sensors (readers) monitoring soil moisture levels in different fields. The challenge is to ensure that:

  • Mutual Exclusion: Only one pump can access the irrigation system at a time to avoid conflicts and ensure efficient water distribution.
  • Progress: The irrigation system should allow pumps to access the system promptly when needed, without being indefinitely blocked by other pumps or sensors.
  • Bounded Waiting: No pump should be starved of access to the irrigation system for an indefinite amount of time due to the continuous reading of sensors.

To implement this, we can model it using the reader-writer problem. Here's how it would work:

  • Readers (Sensors): Sensors continuously monitor soil moisture levels in different fields. They can read the data without interfering with each other. Multiple sensors can read simultaneously without any issues.
  • Writers (Pumps): Pumps are responsible for irrigating the fields based on the readings from the sensors. However, only one pump should be allowed to access the irrigation system at a time to prevent conflicts.

Process Synchronization in Agricultural Irrigation System

        
          #include <iostream>
          #include <thread>
          #include <mutex>
          #include <condition_variable>
          #include <vector>
          #include <chrono>
  
          std::mutex mtx;
          std::condition_variable cv;
          int reader_count = 0;
          bool writer_present = false;
          int soil_moisture = 50; // Initial soil moisture level
  
          void sensor(int id) {
              while (true) {
                  std::unique_lock<std::mutex> lock(mtx);
                  cv.wait(lock, [] { return !writer_present; });
                  reader_count++;
                  lock.unlock();
  
                  // Reading the soil moisture level
                  std::cout << "Sensor " << id << " reads soil moisture: " << soil_moisture << std::endl;
  
                  lock.lock();
                  reader_count--;
                  if (reader_count == 0) {
                      cv.notify_all();
                  }
                  lock.unlock();
  
                  // Simulate the time taken to read the moisture level
                  std::this_thread::sleep_for(std::chrono::milliseconds(100 + rand() % 400));
              }
          }
  
          void irrigation_pump(int id) {
              while (true) {
                  std::unique_lock<std::mutex> lock(mtx);
                  cv.wait(lock, [] { return reader_count == 0 && !writer_present; });
                  writer_present = true;
                  lock.unlock();
  
                  // Updating the soil moisture level
                  soil_moisture += 5; // Simulate irrigation
                  std::cout << "Irrigation pump " << id << " updates soil moisture to: " << soil_moisture << std::endl;
  
                  lock.lock();
                  writer_present = false;
                  cv.notify_all();
                  lock.unlock();
  
                  // Simulate the time taken to irrigate
                  std::this_thread::sleep_for(std::chrono::milliseconds(500 + rand() % 1000));
              }
          }
  
          int main() {
              srand(time(0)); // Seed for random number generation
  
              // Create sensor and irrigation pump threads
              std::vector<std::thread> sensors;
              std::vector<std::thread> pumps;
  
              for (int i = 0; i < 5; ++i) {
                  sensors.push_back(std::thread(sensor, i + 1));
              }
  
              for (int i = 0; i < 2; ++i) {
                  pumps.push_back(std::thread(irrigation_pump, i + 1));
              }
  
              // Join threads (in a real application, you might have a different mechanism to terminate threads)
              for (auto& t : sensors) {
                  t.join();
              }
  
              for (auto& t : pumps) {
                  t.join();
              }
  
              return 0;
          }
